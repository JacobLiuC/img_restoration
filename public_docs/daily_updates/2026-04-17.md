## 当日焦点

- 图像编辑的“规划层”开始独立成型，任务改写和代理执行第一次在同一批公开论文里同时出现。
- 恢复方向也不再只比感知指标，而是加入 UHD 场景、复杂伪影和设备侧实际约束。
- 这一天更像是一个分界点：编辑和恢复都在从单轮模型竞赛，转向系统级工作流竞赛。

## 论文动向

### [From Competition to Coopetition: Coopetitive Training-Free Image Editing Based on Text Guidance](https://arxiv.org/abs/2604.15948v1)

- 发布时间：2026-04-17
- 这篇把训练外图像编辑重新表述为“合作 + 竞争”的控制过程，重点不是再训练一个更大的编辑器，而是提升训练外编辑时的稳定性与指令对齐。
- 它的重要性在于提醒我们：测试时控制策略仍然有很大空间，不一定每次都要靠再训练解决问题。

### [Making Image Editing Easier via Adaptive Task Reformulation with Agentic Executions](https://arxiv.org/abs/2604.15917v1)

- 发布时间：2026-04-17
- 这篇直接把任务改写和 agentic execution 放到标题里，代表图像编辑已经不满足于“给一句 prompt 就结束”。
- 对你们最有价值的点，是它把复杂编辑拆成更容易执行的子任务，这和恢复链路里的 parser 与 planner 非常接近。

### [Diffusion Autoencoder for Unsupervised Artifact Restoration in Handheld Fundus Images](https://arxiv.org/abs/2604.15723v1)

- 发布时间：2026-04-17
- 这篇关注的是无监督伪影恢复，说明恢复任务仍然在向“数据稀缺 + 真实噪声 + 非配对场景”继续推进。
- 它的启发是：恢复链路的鲁棒性不能只靠成对监督数据，真实世界里还要准备无监督或弱监督版本。

### [From Zero to Detail: A Progressive Spectral Decoupling Paradigm for UHD Image Restoration with New Benchmark](https://arxiv.org/abs/2604.15654v1)

- 发布时间：2026-04-17
- 这篇值得关注的不是单一方法名，而是它把 UHD 恢复和新 benchmark 一起推出来。
- 这说明高分辨率恢复已经不再只是把模型挪到更大图上，而是在频谱、细节恢复和评测标准上都要重做。

## 产品与工程

- [Gemini 3.1 Flash Image 模型卡](https://deepmind.google/models/model-cards/gemini-3-1-flash-image/) 代表的仍是“多轮编辑 + 搜索 + 思考”路线，这和当天论文里出现的 agentic execution 形成直接呼应。
- [Nano Banana Pro](https://blog.google/technology/ai/nano-banana-pro/) 继续把图像编辑能力往高一致性与高分辨率方向推，说明产品侧已经默认编辑系统必须兼顾生成能力和稳定控制。
- [Qwen-Image-Edit-2511](https://huggingface.co/Qwen/Qwen-Image-Edit-2511) 和 [StepFun 模型页](https://huggingface.co/stepfun-ai/models) 这条线则继续强化一致性、漂移控制和几何推理。

## 对系统设计的启发

1. parser 不该只产出一句描述，而应该生成一个可执行的子任务序列。
2. planner 不该只选一个 prompt，而要负责“原问题如何改写成更容易执行的问题”。
3. verifier 不能只在最后打分，应该参与中间步骤的重试与回退。

## 今天最值得带走的结论

4 月 17 日最重要的信号，不是又出现了一个更大的编辑模型，而是图像编辑正式开始把 `task reformulation + agentic execution` 当成核心能力来做。这对恢复系统的意义很直接：下一阶段真正拉开差距的，不是谁更会生成，而是谁更会拆任务、做判断并控制步骤。
